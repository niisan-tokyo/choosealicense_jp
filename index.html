  • 単純で寛大なものがいい

    MITライセンス は寛大なライセンスで, 短く的確にまとめられています. このライセンスのもとでは, 誰もがプロジェクトのコードを使ってあらゆることをすることができますが, そのために使用者は コードがあなたに帰属していることを明示し, 保証を求めないことを了承する必要があります

    jQuery や Rails が, この MIT ライセンスを使用しています.

  • 特許についても考慮しておきたい

    The Apache ライセンス は MIT ライセンスに似て寛容なライセンスです. しかし, このライセンスでは 「使用者に対して作成者が特許としての使用許可を出している」という表現を使用しています.

    Apache, SVN, および NuGet が Apache ライセンスを使用しています.

  • コードの改良を共有したい

    GPL (V2 あるいは V3) は, コピーレフトライセンスと呼ばれるもので, コードの再配布もしくは改良物を配布する際には同じライセンスのもとで ソースコードが入手できるようにしなければなりません. V3 は V2 似ているものの, ソフトウェアの変更ができないハードウェアでの使用に対しての制限が追加されています.

    Linux, Git, および WordPress が GPL を使用しています.

  • そもそもコードではない

    クリエイティブ・コモンズを調べてみよう.
    (訳注:OSSのライセンスに似たもので, 著作権を根拠とする使用許諾を与えるものです)

  • 他の選択肢がほしい.

    他のライセンスを参照してみましょう.

  • ライセンスなんか必要ない.

    無理をしてライセンスを選ぶ必要はありません.
